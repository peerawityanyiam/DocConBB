# UAT Checklist ก่อน Go-Live (ระบบติดตามเอกสาร)

## 1) เป้าหมาย
- ยืนยันว่า flow หลักของระบบใช้งานได้จริงในสภาพ production
- ยืนยันว่าแต่ละบทบาทเห็นข้อมูลและทำ action ได้ถูกสิทธิ์
- ยืนยันว่าการอัปโหลดไฟล์ (Word/PDF/ภาพ) เสถียรบนมือถือและคอม

## 2) เงื่อนไขก่อนเริ่ม UAT (Entry Criteria)
- Deploy เวอร์ชัน RC ล่าสุดขึ้น environment ทดสอบแล้ว
- มี user ทดสอบครบทุกบทบาท:
  - `STAFF`
  - `REVIEWER`
  - `DOCCON`
  - `BOSS` (ผู้สั่งงาน)
  - `SUPER_ADMIN`
- เตรียมไฟล์ทดสอบ:
  - Word `.docx` ขนาดเล็ก 1 MB และขนาดกลาง 5-10 MB
  - PDF หลายขนาด
  - รูป 1 รูป / 10 รูป / 20 รูป (รวมใกล้ 80 MB)

## 3) Test Matrix (ตามบทบาท)

## STAFF
- [ ] เห็นเฉพาะงานที่ตัวเองเกี่ยวข้อง
- [ ] อัปโหลด Word แล้วส่งต่อได้
- [ ] เมื่อโดนตีกลับ ต้องกรอกเหตุผลก่อนส่งกลับ
- [ ] แท็บเสร็จแล้ว แสดงเฉพาะงานของตัวเอง

## REVIEWER
- [ ] เห็นเฉพาะงานที่ตัวเองเกี่ยวข้อง
- [ ] ตรวจงานและส่งกลับ/ส่งต่อได้ถูกต้อง
- [ ] เหตุผลตีกลับบังคับกรอก
- [ ] แท็บเสร็จแล้ว แสดงเฉพาะงานของตัวเอง

## DOCCON
- [ ] เห็นงานตามกติกาที่ระบบกำหนดถูกต้อง
- [ ] Checkbox ใน card บันทึกค่าได้จริง (รีเฟรชแล้วยังอยู่)
- [ ] ลบไฟล์แนบได้ตามสิทธิ์ที่กำหนด
- [ ] ปุ่มลิงก์ไปคลังเอกสาร (GAS) เปิดได้

## BOSS (ผู้สั่งงาน)
- [ ] สร้างงานใหม่ได้
- [ ] ลบงานจากหน้ารายละเอียดได้ตามเงื่อนไข
- [ ] ติดตามเฉพาะงานที่ตนเองสั่ง (ถ้าตาม design ล่าสุด)
- [ ] สรุปสถานะบน card ถูกต้อง

## SUPER_ADMIN
- [ ] เข้าหน้าจัดการผู้ใช้ได้
- [ ] ปรับ role ผู้ใช้ได้และมีผลจริง
- [ ] หน้า tracking และภาพรวมเปิดได้ครบ

## 4) Flow ทดสอบสำคัญ (End-to-End)

## F-01 สร้างงานจนเสร็จ
- [ ] BOSS สร้างงานพร้อมไฟล์
- [ ] STAFF/REVIEWER ทำงานตามลำดับจนถึงเสร็จ
- [ ] งานไปอยู่แท็บเสร็จแล้วและข้อมูลครบ

## F-02 ตีกลับและแก้ไข
- [ ] ส่งกลับเพื่อแก้ไขต้องระบุเหตุผล
- [ ] ฝั่งรับเห็นเหตุผลชัดเจน
- [ ] อัปโหลดไฟล์แก้ไขใหม่แล้วส่งต่อได้

## F-03 อัปโหลดรูปหลายรูปแล้วรวม PDF
- [ ] เลือกหลายรูปแล้วแสดงรายการครบทุกรูป
- [ ] ระบบอัปโหลดครบทุกรูปก่อนกด action ต่อ
- [ ] กด action แล้วรวมเป็น PDF ได้
- [ ] หาก fail ต้องแจ้งสาเหตุไทยชัดเจน

## F-04 เปลี่ยนสถานะแล้วไฟล์ PDF เก่าหายจาก card
- [ ] เมื่อส่งต่อ/เปลี่ยนขั้นตอน PDF ที่ไม่ควรแสดงต้องหายจาก card
- [ ] ประวัติไฟล์ยังย้อนดูได้ถูกต้อง

## F-05 สรุปภาพรวม
- [ ] จำนวนงานทั้งหมด/กำลังดำเนินการ/รออนุมัติ/เสร็จสมบูรณ์ ตรง DB
- [ ] แยกตามสถานะและ pipeline ตรงกับข้อมูลจริง

## 5) Performance & Mobile
- [ ] หน้า tracking โหลดเริ่มต้นไม่หน่วงผิดปกติ
- [ ] Mobile ไม่ล้นขวา / ไม่ตัด card
- [ ] Modal สร้างงานใหม่เลื่อนได้ครบ

## 6) Exit Criteria (เงื่อนไขพร้อม Go-Live)
- [ ] Test case critical ผ่าน 100%
- [ ] Bug ระดับบล็อก (`P0/P1`) = 0
- [ ] Bug `P2` ต้องมีแผนแก้และ owner ชัดเจน
- [ ] มี rollback plan และคนรับผิดชอบช่วง go-live

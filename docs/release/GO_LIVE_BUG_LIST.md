# Bug List ก่อน Go-Live

สถานะที่แนะนำ:
- `OPEN` = ยังไม่แก้
- `VERIFY` = แก้แล้ว รอทดสอบยืนยัน
- `CLOSED` = ทดสอบผ่านแล้ว

ความรุนแรง:
- `P0` = บล็อก go-live ทันที
- `P1` = กระทบงานหลัก ต้องปิดก่อน go-live
- `P2` = กระทบรอง รับได้ถ้ามี workaround

## รายการตรวจซ้ำก่อนปิด bug
- [ ] มีหลักฐานทดสอบจริง (มือถือ + คอม)
- [ ] มีภาพ/วิดีโอประกอบกรณี fail
- [ ] มี log request id จาก API ฝั่งที่ fail
- [ ] ทดสอบ regression flow หลัก: สร้างงาน -> ส่งต่อ -> ตีกลับ -> เสร็จ
